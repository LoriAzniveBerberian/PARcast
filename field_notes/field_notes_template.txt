================================================================
INSTRUMENT OVERVIEW — READ BEFORE FIRST DEPLOYMENT
================================================================
The Teensy runs automatically every time it gets power.
You DO NOT need a laptop, Arduino IDE, or any computer in the field.

STARTING A DEPLOYMENT:
  1. Plug USB power bank into Teensy micro USB port
  2. Wait 3 seconds for startup
  3. Remove PAR sensor cap
  4. Logging is now active — a new CSV file is being written

STOPPING A DEPLOYMENT:
  1. Replace PAR sensor cap
  2. Unplug USB power bank
  3. Deployment complete — CSV file is saved automatically

STARTING A NEW DEPLOYMENT (same day):
  1. Simply unplug and replug the power bank
  2. A new file is created automatically (e.g. 0001 → 0002)
  3. No laptop needed

THE RESET BUTTON — EMERGENCY ONLY:
  The small button on the Teensy board is the reset button.
  DO NOT press it during a deployment.
  It creates a new file and interrupts logging.
  Only press if the Teensy freezes or stops responding.
  If pressed accidentally — note it in the log below immediately.

================================================================
TROUBLESHOOTING
================================================================
Timestamps show 2018:
   - RTC coin cell (CR1220) lost power or unseated
   - Fix: reseat coin cell + side up, re-upload rtc_test.ino to reset time
   - Do NOT use data with 2018 timestamps

Multiple files created unexpectedly:
   - Reset button was pressed accidentally
   - Note which files are affected and at what times
   - Data is still usable — just fragmented

SD card not detected:
   - Push card in firmly until it clicks
   - Check FAT32 MBR format (see SD_CARD_FORMAT_INSTRUCTIONS.txt)
